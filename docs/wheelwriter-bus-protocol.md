# Протокол шины IBM Wheelwriter

Анализ на основе проекта [IBM_Wheelwriter_Teletype](https://github.com/jim11662418/IBM_Wheelwriter_Teletype) (Jim Loos).

## Физический уровень

- **187 500 бод**, 9 бит данных + start + stop = 11 бит на слово
- Инвертированный open-collector (idle = high, данные тянут low)
- Одна шина для обоих направлений (полудуплекс)
- Напряжение шины: 5V (нужен level shifter для Teensy 3.3V)

## Логический уровень

- Все команды начинаются со слова `0x121` (9 бит: bit8=1, byte=0x21)
- За `0x121` следует код команды, затем параметры
- После каждого слова отправитель ждёт, пока шина вернётся в high
- Получатель отвечает ACK: `0x000` (все 9 бит = 0)

## Команды Function Board → Printer Board (клавиатура → печать)

| Последовательность | Значение |
|---|---|
| `0x121, 0x003, <printwheel_code>, <microspaces>` | Нажата буква/символ |
| `0x121, 0x005, <direction\|lines>` | Вертикальное движение (bit7=направление) |
| `0x121, 0x006, <direction>, <microspaces>` | Горизонтальное движение (Space/Backspace/Tab) |
| `0x121, 0x00E, <code>` | Code+клавиша (управляющие символы) |
| `0x121, 0x001` | Команда сброса (reset) |
| `0x121, 0x007` | Прокрутка печатного колеса (spin) |
| `0x121, 0x004, <printwheel_code>, <microspaces>` | Печать на корректирующей ленте (erase) |

### Печать символа: `0x121, 0x003, <pw_code>, <microspaces>`

- `pw_code` — позиция символа на печатном колесе (см. таблицы ниже)
- `microspaces` — сколько микрошагов сдвинуть каретку вправо после печати
  - 10 микрошагов = 1 символ при 12cpi (Elite)
  - 12 микрошагов = 1 символ при 10cpi (Pica)
  - 8 микрошагов = 1 символ при 15cpi (Micro Elite)

### Горизонтальное движение: `0x121, 0x006, <dir_hi>, <count_lo>`

- 11-битное число микрошагов: верхние 3 бита в `dir_hi[2:0]`, нижние 8 в `count_lo`
- `dir_hi` bit7: 1 = вправо, 0 = влево

### Вертикальное движение: `0x121, 0x005, <dir_lines>`

- bit7: 1 = бумага вверх, 0 = бумага вниз
- bits[4:0]: количество микролиний
  - 16 микролиний = 1 строка при 10/12cpi
  - 12 микролиний = 1 строка при 15cpi

### Code+клавиша: `0x121, 0x00E, <code>`

- `code & 0x7F` определяет клавишу (bit7 различается на WW3 vs WW6)
- `0x067` = клавиша Code отпущена

## Ответ Printer Board на reset

После `0x121, 0x001` Printer Board отвечает кодом установленного печатного колеса:

| Ответ | Колесо |
|---|---|
| `0x008` | PS (Proportional Spacing) |
| `0x010` | 15P (15 cpi) |
| `0x020` | 12P (12 cpi) |
| `0x040` | 10P (10 cpi) |
| `0x021` | Колесо не установлено |

## Применение к нашему проекту (Teensy 4.0)

Референсный проект использует MCU с 4 UARTами и **перехватывает** шину (сидит между Function Board и Printer Board).

Наша задача проще — подключение через **Option Interface** порт (та же шина). Нужен:
- Один UART Teensy на 187500 бод, 9-bit mode: `Serial1.begin(187500, SERIAL_9N1)`
- Level shifter 3.3V ↔ 5V (open-collector/open-drain)
- Инверсия сигнала (шина инвертирована)

Для чтения клавиш: слушаем шину и декодируем последовательности через стейт-машину (как `ww_decode_keys`).

Для отправки на печать: формируем команды `0x121, 0x003, ...` и ждём ACK после каждого слова.
